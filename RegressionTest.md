Objective

This activity aims to help you:

• Apply Vertical Slice Architecture in restructuring your existing multi-platform software project.
• Improve the maintainability, modularity, and scalability of your system architecture.
• Develop a comprehensive Software Test Plan for all system requirements.
• Perform a Full Regression Test to validate that all functional requirements still work after refactoring.
• Practice proper software engineering workflows using Git branching, refactoring, testing, and documentation.

Preconditions
Before proceeding with this activity:

• Merge ALL remaining completed tasks/features into the main branch.
• Ensure the main branch is updated, stable, and free from merge conflicts.

Task
Part 1 — Branch Creation
Create a new branch from the updated main branch.

Requirements:
• Branch name must be meaningful and related to the task.
• All refactoring work must be done in this new branch.

Part 2 — Vertical Slice Refactoring
Refactor the entire project structure to apply Vertical Slice Architecture.

Scope:
• Backend
• Web Frontend
• Mobile Application (if applicable)

Requirements:
• Organize the project by feature/module rather than by technical layers.
• Ensure all existing features remain functional after refactoring.
• Maintain clean code and proper naming conventions.

Part 3 — Test Plan Creation
Create a complete Software Test Plan covering all system requirements.

The Test Plan must include:

• Functional Requirements Coverage
• Test Cases
• Test Scripts / Test Steps
• Automated Test Cases

Requirements:
• Cover all implemented functional requirements.
• Include clear testing procedures.
• Include automated tests for major modules/features.

Part 4 — Full Regression Testing
Perform a Full Regression Test for the entire system.

Coverage must include all implemented functional requirements.

Requirements:
• Validate that all features are working correctly after refactoring.
• Record all test results.
• Identify any bugs or regressions encountered.

Part 5 — Full Regression Test Report
Prepare and submit a complete Regression Test Report.

The report must include:

• Project Information
• Refactoring Summary
• Updated Project Structure
• Test Plan Documentation
• Automated Test Evidence
• Regression Test Results
• Issues Found
• Fixes Applied
